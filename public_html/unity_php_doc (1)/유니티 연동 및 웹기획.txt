1. 서버 설계

- 필요기능

	1. 회원가입

		아이디, 패스워드 입력

		# 회원 테이블 설계

	2. 회원인증

		인증후 세션키 발행
		세션키 인증 기능 (세션키 유효 시간은 500초)
		세션키의 재발행 기능

		# 로그인 세션 테이블 설계


	3. 스테이지클리어

		클리어 스테이지의 저장 기능
		반복 클리어 여부 체크 기능

		# 클리어 스테이지 테이블 설계
		# 스테이지 데이터 테이블 설계



	4. 스테이지클리어 경험치 보상

		경험치에 따른 레벨업 기능

		# 경험치,레벨 설계
		# 스테이지 데이터 테이블 + 경험치  (stage 클리어 지급 경험치)
		# 레벨업 데이터 테이블 (레벨별 레벨상승 경험치)
		

	5. 나의 정보 얻기

		내 ID 와 경험치,레벨 얻음.



	+. ResultCode 와 ResultMessage 디파인 및 메시지 스트링의 정리

		모든 결과의 ResultCode 와 ResultMsg 부분에
		디파인 된 코드와 스트링을 사용함.

		샘플) _ResultCode.php

		예시) QuitError(df_RESULT_DB_QUERY_FAIL);

			위 처럼 사용시 df_RESULT_DB_QUERY_FAIL 에러코드와
			해당 에러 메시지를 JSON 형태로 보내고 종료 함.
